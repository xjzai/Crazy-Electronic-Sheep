# 《电子羊会发疯》MVP试玩版实施计划

## 1. 文档目的

本文件是当前仓库内关于 `2 张地图、25 只羊、单货币 MVP 试玩版` 的唯一实施口径。  
它覆盖当前正式要落地的 MVP 试玩闭环，不再沿用早期“单地图基础闭环子阶段”的临时假设。

本文件面向 AI 开发者与工程实现者，要求按顺序推进，并以本文的冻结决策为准。  
如果其他文档仍保留旧范围描述，以本文和同步后的核心文档为准。

## 2. 本轮正式范围

### 2.1 包含

- 两张地图：`map_01` 与 `map_02`
- 25 只羊：`001-025`

- 自动产出
- 点击查岗

## 3. 本轮冻结决策

### 3.1 技术与测试冻结

- 当前实现按 `Cocos Creator 3.8.x + TypeScript + Node.js LTS + pnpm` 落地。
- 当前实现按 `Vitest` 编写纯逻辑自动化测试。
- 当前实现不要求广告 SDK 真接入，但必须有清晰的广告接口抽象和 mock 成功路径。

### 3.2 内容边界冻结

- 第一张地图内容范围固定为 `001-020`。
- 第二张地图内容范围固定为 `021-025`。
- 本阶段图鉴固定展示 `001-025`，不展示 `026+`。
- `025` 是本阶段最高羊。
- `025 + 025` 不允许继续合成，必须给出明确失败反馈，且无副作用失败。

### 3.3 新档初始状态冻结

- 新档固定赠送 `1` 只 `001`。
- 新档默认进入 `map_01`。
- 新档 `最高解锁羊编号 = 001`。
- 新档 `已解锁羊编号列表 = {001}`。
- 新档图鉴中 `001` 初始为已解锁。
- 新档初始资源由配置控制，当前默认建议值为 `0`，但仍保留为可调字段。

### 3.4 地图与切换冻结

- 两张地图独立摆放羊群，独立维护地图内实例状态。
- 两张地图的自动产出统一汇总到全局资源池。
- 当前显示地图运行完整可视表现：随机走动、拖拽、点击反馈、合成反馈。
- 非当前显示地图继续贡献自动产出，但不持续运行可视随机走动表现。
- 第二张地图首次解锁时，必须自动跳转一次。
- 第二张地图解锁后，玩家可通过明确的地图切换入口手动切换。
- 只要第二张地图已解锁，重进游戏默认进入当前最高已解锁地图；在本阶段即默认进入 `map_02`。

### 3.5 `020 -> 021` 跨地图解锁冻结

- `2只020 -> 021` 是第一张地图到第二张地图的正式解锁门槛。
- 合成成功后，结果羊 `021` 最终必须生成到第二张地图。
- 第一次触发时，必须播放完整转场序列：
  - 先在第一张地图上显示合成结果 `021`
  - `021` 以比正常显示略大一圈的强调尺寸短暂出现
  - 第一张地图整体缩小直至消失
  - `021` 从强调尺寸回落到正常显示尺寸
  - 背景切换到第二张地图
  - `021` 在第二张地图的合法出生位置落定
- 后续再次在第一张地图合成出 `021` 时，不再播放完整解锁演出，但结果仍必须生成到第二张地图。

### 3.6 购买与招聘机冻结

- 购买入口始终作用于“当前正在查看的地图”。
- 当前 MVP 使用最小科技集：
  - `精神稳定仪`
  - `查岗雷达`
  - `黑心招聘机`
  - `离线摸鱼服务器`
- `黑心招聘机` 只有一套全局等级。
- 购买公式固定为：

```text
可购买羊等级 = max(当前地图起始羊, 全局最高解锁羊 H - 当前招聘机偏移 N)
```

- 招聘机等级对应购买偏移固定为：
  - `Lv.0 -> H-5`
  - `Lv.1 -> H-4`
  - `Lv.2 -> H-3`
  - `Lv.3 -> H-2`
- 不开放永久 `H-1` 购买权限。
- 即使公式按全局 `H` 计算，最终购买结果仍必须受当前地图内容上限截断：
  - 第一张地图最多买到 `020`
  - 第二张地图最多买到 `025`

### 3.7 走动与拖拽冻结

- 羊在地图内只有“待机”和“行走”两种基础状态。
- 羊在待机结束后，选择新的合法目标点并直线移动。
- 羊为单张贴图，向左移动时水平翻转，向右移动时恢复原向。
- 拖拽开始后，只有被拖拽羊立即暂停自主走动。
- 其他羊不强制暂停，继续按各自节奏移动。
- 被拖拽羊必须提升到当前地图拖拽显示层最上层。
- 当被拖拽羊接近某只同等级羊并进入合成命中半径时，该目标进入“可合成锁定”状态，必须有明显锁定提示。

### 3.8 合成命中冻结

- 合成判定使用“中心点 + 命中半径”规则。
- 释放瞬间，如果被拖拽羊中心点进入某只同等级目标羊的合成命中半径，则触发合成。
- 如果同时命中多个同等级目标，优先选择“中心点距离最近”的目标。
- 未命中时，不回原位；应落到释放点附近最近合法位置。

### 3.9 一键合成冻结

- 一键合成免费使用。
- 一键合成冷却固定为 `60 秒`。
- 一键合成只作用于当前正在查看的地图。
- 一键合成执行规则固定为“从低到高递归清扫”：
  - 先尽可能合成当前地图中的低等级对
  - 新生成的羊若又能继续合成，也必须继续参与递归
  - 一直执行到当前地图中再也没有可合成对为止

### 3.10 点击与加速冻结

- 单羊查岗冷却固定为 `5 秒`。
- 点击收益基于该羊当前秒产计算。
- `超频摸鱼` 固定为：自动产出 `×2`，持续 `60 秒`，每天 `3 次`。
- `超频摸鱼` 只作用于自动产出，不作用于点击查岗收益。
- `超频摸鱼` 不进入离线收益结算基数。

### 3.11 离线收益冻结

- 离线收益只基于“退出或切后台时保存的基础全局总秒产快照”结算。
- 该快照必须排除所有临时加成和限时效果，包括：
  - `超频摸鱼`
  - 收益翻倍类道具
  - 广告临时加成
  - 其他短时 Buff
- 离线收益弹窗必须展示：
  - 离线时长
  - 结算所用基础全局秒产
  - 普通领取金额
  - 广告 `3 倍` 领取金额
- 玩家必须在普通领取和广告 `3 倍` 领取之间二选一。
- 广告 `3 倍` 领取每天限 `1 次`。
- 当天已用掉 `3 倍` 后，后续离线收益只能普通领取。
- 没有可结算离线时长时，不弹离线奖励面板。

### 3.12 存档冻结

- 存档只保存长期进度真值，不保存临时 Buff 或临时冷却剩余时间。
- 明确不保存：
  - `超频摸鱼` 剩余时间
  - 一键合成剩余冷却
  - 临时表现状态
- 重进后，上述临时状态一律按“未激活”处理。
- 当前阶段坏档恢复策略暂定为：
  - 只要关键结构损坏或关键字段非法，就放弃旧档并重建新档
  - 必须明确提示用户“存档损坏，已为你恢复新进度”
  - 该策略是当前实现冻结口径，后续允许单独评审是否升级为部分恢复

### 3.13 转场与交互锁冻结

- 地图切换转场期间，玩家交互全部锁定。
- 转场期间不可拖拽、不可点击羊、不可购买、不可再次切图。
- 转场期间自动产出业务真值继续正常推进。
- 转场期间可视走动表现暂停，转场结束后恢复。

### 3.14 自动产出刷新冻结

- 自动产出按严格“每秒结算一次”处理。
- HUD 数值按秒跳动，不做平滑连续累计。

## 5. 分步实施顺序

### 步骤 1：锁定工程版本与目录结构

目标：
冻结本轮 MVP 的引擎版本、脚本语言、目录结构和测试入口。

指令：
- 明确 `Cocos Creator 3.8.x`、`TypeScript`、`Node.js LTS`、`pnpm`。
- 建立场景、脚本、配置、测试、UI、素材目录。
- 建立 `Vitest` 逻辑测试入口。

验证测试：
- 项目能打开并本地预览。
- 测试命令能运行空测试集。

### 步骤 2：建立双地图主场景骨架

目标：
让主场景一开始就具备双地图 MVP 的结构，而不是单地图临时结构。

指令：
- 建立启动场景与主游戏场景。
- 主游戏场景预留：顶部 HUD、地图区域、底部操作区、地图切换入口、图鉴入口、科技入口、调试入口。
- 第二张地图未解锁时，切换入口应明确显示未解锁状态。

验证测试：
- 启动后先进入启动场景，再进入主游戏场景。
- 主界面能看到地图区域与切换入口占位。

### 步骤 3：建立 MVP 配置文件与校验

目标：
让配置足以支撑 `001-025`、双地图、科技和外围系统。

指令：
- 配置至少包含：
  - `001-025` 羊配置
  - 两张地图的起始羊与上限羊
  - 地图边界与容量
  - 秒产、基础购买价、点击倍率
  - 招聘机等级与偏移
  - 科技配置
  - 一键合成冷却
  - 超频摸鱼参数
  - 离线上限与广告三倍规则
  - 新档初始资源
- 启动时先校验配置，再进入主场景。

验证测试：
- 正确配置能通过启动。
- 删除关键字段时能明确报错。

### 步骤 4：建立全局状态与新档初始化

目标：
把双地图 MVP 的唯一业务真值固定下来。

指令：
- 状态至少包含：
  - 全局资源
  - 全局最高解锁羊
  - 已解锁羊列表
  - 图鉴状态
  - 招聘机等级与科技等级
  - 每张地图的羊实例列表
  - 第二张地图解锁状态
  - 当日超频使用次数
  - 当日离线 `3 倍` 使用次数
  - 上次退出时间与基础全局秒产快照
- 新档按本文第 `3.3` 节初始化。

验证测试：
- 新档启动后固定拥有 `1` 只 `001`。
- 新档图鉴中 `001` 为已解锁。
- 第二张地图初始为未解锁。

### 步骤 5：实现双地图模型与容量约束

目标：
建立 `map_01` 与 `map_02` 的独立地图状态。

指令：
- 每张地图独立维护：
  - 漫游边界
  - 活动羊容量
  - 当前地图内羊实例
  - 购买起始羊与上限羊
- 当前显示地图完整运行。
- 非当前地图仅继续自动产出，不跑可视走动。

验证测试：
- 两张地图状态可独立存在。
- 切走后隐藏地图仍继续贡献自动产出。

### 步骤 6：实现羊实例与随机走动

目标：
让 `001-025` 的羊实例能在各自地图中合法生成和移动。

指令：
- 建立羊实例模型与地图归属字段。
- 实现待机、选点、移动、到达状态机。
- 实现左右翻转。

验证测试：
- 当前地图中的羊能持续随机走动。
- 非当前地图切回后能恢复可视走动。

### 步骤 7：实现自动产出主循环

目标：
让双地图总秒产按秒驱动全局资源增长。

指令：
- 每秒结算一次全局资源增长。
- 全局总秒产 = 两张地图当前所有羊实例秒产之和，再叠加长期科技加成。

验证测试：
- 地图一与地图二同时有羊时，总资源按合计秒产增长。
- 超频未开启时，点击收益不受自动产出逻辑影响。

### 步骤 8：实现招聘机与科技系统

目标：
让购买范围和最小科技集进入正式业务闭环。

指令：
- 实现四项科技：
  - `精神稳定仪`
  - `查岗雷达`
  - `黑心招聘机`
  - `离线摸鱼服务器`
- `黑心招聘机` 使用本文固定公式。
- 购买范围必须同时满足：
  - 全局 `H-N`
  - 当前地图起始羊下限
  - 当前地图内容上限

验证测试：
- 提升招聘机等级后，两张地图的可购买范围都发生预期变化。
- 第一张地图不会买出 `021+`。
- 第二张地图不会买出 `026+`。

### 步骤 9：实现购买系统

目标：
让玩家能在当前地图中购买合法范围内的羊。

指令：
- 购买入口始终绑定当前地图。
- 资源不足、地图满员、购买范围非法都必须明确失败。
- 成功购买后，羊生成到当前地图合法出生点。

验证测试：
- 在第一张地图购买只影响第一张地图。
- 在第二张地图购买只影响第二张地图。
- 第一图满员不影响第二图继续购买。

### 步骤 10：实现拖拽、目标锁定与两两合成

目标：
让拖拽合成在移动目标下仍然稳定可用。

指令：
- 被拖拽羊暂停，其他羊不停。
- 接近同等级目标时出现锁定提示。
- 使用中心点 + 命中半径判定。
- 多目标命中时选择最近目标。
- 未命中则回落到最近合法位置。

验证测试：
- 拖拽过程能稳定锁定目标。
- 其他羊继续移动时，仍可顺利完成合成。

### 步骤 11：实现跨地图边界合成与第二张地图解锁

目标：
打通 `020 -> 021` 的世界切换体验。

指令：
- 实现 `020 + 020 -> 021`。
- 第一次触发时按本文固定转场序列播放。
- 第二次及以后继续在第一图合成出 `021` 时，结果直接进入第二图，不再播放完整解锁演出。
- 第二图解锁后，自动跳转一次，之后开放手动切换。

验证测试：
- 首次 `020 + 020` 后，第二张地图解锁并自动跳转。
- `021` 最终在第二张地图落定。

### 步骤 12：实现第二张地图 `021-025` 推进与 `025` 封顶

目标：
让第二张地图形成完整的 `021-025` 闭环。

指令：
- 第二张地图购买范围、合成、点击、产出都按当前地图规则独立运作。
- `025 + 025` 必须明确失败。

验证测试：
- 第二图可正常推进到 `025`。
- `025 + 025` 不继续升级，且无副作用失败。

### 步骤 13：实现一键合成与超频摸鱼

目标：
补齐当前 MVP 保留的外围系统。

指令：
- 一键合成按当前地图递归清扫。
- 超频摸鱼只放大自动产出，不放大点击收益，不进入离线结算。

验证测试：
- 一键合成能一直执行到当前地图无可合成对为止。
- 超频开启后，自动产出翻倍，点击收益不翻倍。

### 步骤 14：实现图鉴、HUD 与地图切换动画

目标：
让玩家能清楚看见双地图当前进度。

指令：
- 图鉴固定展示 `001-025`。
- HUD 至少显示：
  - 当前资源
  - 全局总秒产
  - 全局最高解锁羊
  - 图鉴进度
  - 当前地图标识
- 地图切换动画期间锁定交互，但自动产出继续。

验证测试：
- 第一图与第二图切换动画符合冻结口径。
- 重进后若已解锁第二图，默认进入第二图。

### 步骤 15：实现本地存档、坏档恢复与离线收益

目标：
让关闭重进后的核心进度与离线体验成立。

指令：
- 存档至少保存：
  - 全局资源
  - 全局最高解锁羊
  - 已解锁羊列表
  - 图鉴状态
  - 科技等级
  - 每张地图羊实例列表
  - 第二图解锁状态
  - 每日超频次数
  - 每日离线 `3 倍` 次数
  - 上次退出时间
  - 退出时基础全局总秒产快照
